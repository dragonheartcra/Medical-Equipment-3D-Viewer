## 2. 项目目标和需求边界

需求来源见：

- `需求文档-医学设备3D交互展示.md`
- `项目交接文档-完整版.md`

用户多次确认后，当前实现范围是“简单但精致”的 3D 教学展示页，不做复杂结构浏览器。

核心需求：

- 5 台设备：DR、CT、DSA、MRI、超声。
- 每台设备展示一个整体 3D 模型。
- 模型可旋转、缩放。
- 模型上放少量关键标注点。
- 鼠标悬浮或点击标注点后，右侧显示部件图片和说明。
- 细节图用现有 2D 教学图片，不额外制作部件 3D。

视觉要求：

- 参考用户提供的深色科技风/三星堆文物展示风格。
- 深色背景、蓝色高亮、精致面板、暗角聚焦效果。
- 不复刻复杂功能，例如材料信息、仿真分析、剖切、隐藏部件、底部大量工具栏。

## 6. 当前交互实现细节

### 6.1 设备切换

左侧卡片列表切换 5 台设备：

```text
DR / CT / DSA / MRI / 超声
```

切换时：

- 更新页面标题和简介。
- 更新右侧详情默认部件。
- 重新渲染标注点。
- 卸载旧模型。
- 加载新模型。

### 6.2 标注点

当前标注点不是 GLB 内部的一部分，而是 HTML 层叠在 canvas 上。

实现方式：

- `src/devices.js` 中保存 3D 坐标和可选表面法线。
- `src/main.js` 每帧用 `Vector3.project(camera)` 将 3D 坐标投影到屏幕。
- 在 `.hotspot-layer` 中移动对应 DOM 按钮。
- 新调试模式中点选出来的标注点使用模型局部坐标，显示时通过 `activeModel.localToWorld()` 转回世界坐标。
- 如果部件配置有 `normal`，标注点会沿表面法线略微浮出，避免埋进模型表面。

优点：

- 不依赖 Tripo 模型是否拆 mesh。
- 标签样式更容易做精致。
- 调试模式中可以直接点击模型表面取点。

当前标注点视觉：

- 蓝色圆点表示模型上的实际标记位置。
- 右侧白色文字框可以点击。
- 已移除圆点和文字框之间的连线，因为线条无法保证对准模型表面，容易造成误导。

### 6.3 详情面板

右侧面板显示：

- 当前部件名称。
- 当前部件说明。
- 当前层级按钮。
- 当前层级图片。
- 当前层级说明。

点击同一个标注点：

- 如果有多层图，会向下一层切换。
- 到最后一层后停留在最后一层。
- 普通模式下会触发 3D 镜头聚焦动画，平滑移动到对应部件位置附近。
- 用户拖动模型或点击“重置视角”会取消未完成的聚焦动画。

### 6.4 图片放大

## 7. 标注点调试模式

访问：

```bash
http://localhost:5174/?debug=1
```

功能：

- 页面右下角显示“标注点调试”面板。
- 可以选择当前要定位的部件。
- 点击“开始定位”后，鼠标移动到模型表面会显示蓝色预览点。
- 单击模型表面后，用 Three.js `Raycaster` 获取真实表面交点。
- 保存交点为模型局部坐标，同时保存表面法线 `normal`。
- 面板中显示当前设备的所有标注点坐标和法线。
- 可以点击“复制当前坐标”复制文本。

用户的预期流程：

1. 用户打开 `?debug=1`。
2. 用户选择设备和部件。
3. 用户先旋转、缩放模型到合适角度。
4. 用户点击“开始定位”。
5. 用户在模型表面目标位置单击。
6. 用户截图或复制右下角坐标给 agent。
7. agent 将这些坐标和法线写回 `src/devices.js`。
8. 普通模式 `/` 会使用新坐标。

调试面板输出示例：

```text
dr
selected: xray-tube [0.180, -0.010, 0.052]

  // X线球管
  id: "xray-tube",
  position: [0.180, -0.010, 0.052],
  normal: [0.999, -0.035, -0.017],
```

注意：

- 旧的“拖动 HTML 标注点”方案已经废弃，因为旋转模型后会出现视觉漂移。
- 现在的取点以 Raycaster 打到的第一个模型表面为准。
- 用户应先把目标表面转到可见位置，再单击取点。
- 拖动鼠标仍然只用于旋转模型，不会误保存坐标。


主要参数：

### 12.3 GLB 压缩

当前未压缩。

原因：

- 用户要求先看视觉，不急着压缩。
- 模型原始质量需要先确认。

后续可考虑：

- 使用 gltf-pipeline + Draco。
- 或使用 gltf-transform / gltfpack。

注意：

- 如果使用 Draco 压缩，前端需要重新加入 `DRACOLoader`。
- 之前曾短暂加入过 Draco 支持，但因为当前不压缩，会增加构建体积，所以已移除。
- 真正压缩时需要重新加回 Draco 解码器，并确认 `public/draco/` 或打包路径正确。
- **压缩不会影响视觉效果**，Draco 只压缩几何顶点数据，解压后渲染效果与原始一致。
