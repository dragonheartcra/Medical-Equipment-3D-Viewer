# 医学影像设备3D交互展示 — 项目交接文档

> **项目目标**：建立5台医学影像设备的3D交互展示网页（DR、CT、DSA、MRI、超声），整体3D模型可旋转缩放，关键部件支持悬浮/点击查看详情图。

> **文档定位**：这是“历史全量交接文档”，保留早期素材、参考项目和技术设想。它不是当前最新实现说明，后续接手不要直接按本文档里的 React/R3F 或极限压缩方案执行。

## 文档关系与阅读顺序

当前项目有 3 个主要文档：

1. `当前开发交接文档-医学设备3D展示.md`：最新接手入口，记录当前代码实际状态、普通版/调试版关系、Raycaster 表面取点、已确认标注点、已实现镜头聚焦效果和后续建议。
2. `需求文档-医学设备3D交互展示.md`：需求基准，只用于核对客户原始要求、设备部件和图片层级。
3. `项目交接文档-完整版.md`：历史资料归档，保留项目早期路径、素材来源、参考仓库和曾经考虑过的技术路线。

当前实际实现已经改为 `Vite + 原生 HTML/CSS/JavaScript + Three.js`，不是 React，也暂未压缩 GLB。以 `当前开发交接文档-医学设备3D展示.md` 为准。

## 一、项目背景

### 1.1 需求来源
源文件：`说明.doc`（WPS格式，用 antiword 提取文本）。

## 二、技术方案

### 2.1 核心路线（已确定）
参考来源：**XinGPT (@xingpt) 2025年5月15日推文** + **cell-architecture-studio 开源项目**

```
步骤1：AI生成平面图（GPT Image-2 / 即梦）
    │  每台设备4个角度（正面、左侧、右侧、背面）
    │  喂参考图 + Prompt → 出统一风格的3D渲染风格平面图
    ▼
步骤2：图生3D（Tripo，闲鱼买会员）
    │  4张图丢Tripo → 生成 GLB 3D模型
    ▼
步骤3：压缩GLB（gltf-pipeline + Draco）
    │  目标：单个GLB < 2MB
    ▼
步骤4：网页集成（React Three Fiber + Drei）
    │  克隆 cell-architecture-studio → 替换模型 → 加标注交互系统
    ▼
步骤5：部署（Vite build → 静态托管）
```

### 2.2 参考开源项目
以下两个仓库是技术参照，cell-architecture-studio 最接近我们需求：

| 仓库 | 用途 | 关键信息 |
|------|------|----------|
| https://github.com/cclank/cell-architecture-studio | **直接复刻模板** | React + Three.js (React Three Fiber + Drei)，7个细胞模型切换，带标注详情面板、对比模式。我们替换模型和数据结构就行。 |
| https://github.com/huangserva/3DCellForge | 工具平台参考 | 完整的3D生成工作台，支持混元/Tripo等多后端。技术栈：Next.js + Three.js。偏"工具"而非"展示"。 |

关键源码文件（cell-architecture-studio）：
- `src/App.tsx` — 主入口，Canvas + 设备选择器
- `src/components/CellScene.tsx` — 3D场景，`useGLTF("/models/xxx.glb")` 加载模型
- `src/components/CellModel.tsx` — 模型高亮/标注逻辑
- `src/data/cells.ts` — 数据定义（替换为5台设备）
- `public/models/` — 放GLB文件

### 2.3 为什么不用现成3D模型
搜过 Sketchfab、TurboSquid、CGTrader、NIH 3D 等平台，结论：
- **医疗设备3D模型基本全是付费工业CAD**（$49-$99/个），无免费的网页适配版本
- 少数免费的STL/OBJ无贴图、面数高，不适合网页展示
- **图生3D是最快路径**

## 五、下一步操作清单（给接手的人）

### 阶段一：生成平面图（当前需要立即执行）

**工具**：GPT Image-2 或 即梦（需要支持"参考图+文字"模式）

**方法**：每台设备先生成正面图，再以正面图为参考生成左/右/背。

**Prompt清单**：详见 `平面图生成Prompt清单.md`（共20条，5台设备×4角度）

**参考图对照表**：

| 设备 | 正面参考图 | 左/右/背参考图 |
|------|-----------|---------------|
| DR | `参考图/01_DR_悬吊式DR整机_1160x627.jpg` | 用第1步AI出的正面图 |
| CT | `参考图/02_CT_CT机3_nximg_768x1024.jpg` | 用第1步AI出的正面图 |
| DSA | `参考图/03_DSA_DSA整机1_1440x1080.jpg` | 用第1步AI出的正面图 |
| MRI | `参考图/04_MRI_MRI西门子_2000x1333.jpg` | 用第1步AI出的正面图 |
| 超声 | `参考图/05_US_超声整机1_2784x1856.jpg` | 用第1步AI出的正面图 |

**验证策略**：先跑DR一台，4张图丢Tripo出GLB，效果OK再走剩下4台。

### 阶段二：图生3D（Tripo）

1. 闲鱼买Tripo会员（约￥20/月）
2. 每台设备上传4张AI生成的平面图 → 生成GLB
3. 下载GLB文件

### 阶段三：压缩GLB

```bash
# 安装工具
npm install -g gltf-pipeline

# 压缩（Draco压缩，级别7）
npx gltf-pipeline -i input.glb -o output.glb --draco.compressMeshes --draco.compressionLevel 7
# 目标：单个文件 < 2MB
```

### 阶段四：网页开发

1. **克隆模板**：`git clone https://github.com/cclank/cell-architecture-studio.git`
2. **替换模型**：将压缩后的GLB放入 `public/models/`
3. **修改数据**：编辑 `src/data/cells.ts`，替换为5台设备的数据结构（名称、描述、GLB路径、标注点坐标、关联的细节图）
4. **加标注系统**：在3D模型上放透明小球（SphereGeometry+MeshBasicMaterial），Raycaster检测hover/click，触发侧边栏显示对应细节图
5. **细节图引用**：侧边栏显示的图片使用 `素材图片/` 目录中的文件

### 阶段五：部署

```bash
npm install
npm run build
# dist/ 目录部署到任意静态托管
```

## 六、关键技术要点

### 6.1 3D颜色哪来的
GLB的颜色是图生3D时自动烘焙上去的——你喂的参考图长什么颜色，模型就出什么颜色。不需要额外上色。

### 6.2 标注交互怎么做
混元3D/Tripo出的GLB大概率是单mesh（所有部件融合成一块）。不能在mesh级别区分部件。做法：
- 在3D模型上手动放置不可见标注小球（指定坐标 [x,y,z]）
- Raycaster 检测小球 hover → 显示 tooltip
- Raycaster 检测小球 click → 显示细节图

### 6.3 GLB文件大小
- 网页加载要求单文件 < 2MB
- 混元3D出的GLB往往偏大，必须用 gltf-pipeline + Draco 压缩

### 6.4 细节图不需要3D
说明.doc中规定的细节展示直接用现有2D图片（`素材图片/` 目录），不需要每个部件都做3D模型。只有5台设备整体需要3D。

## 七、注意事项

1. **原始素材图片不能直接用于图生3D** — 教材插图有中文标注、箭头引线，AI会把文字当成立体结构建模。必须先用AI重新生成干净平面图。
2. **不追求每个部件都3D化** — 说明.doc只要求"整体3D + 点击弹出细节图"，细节图用现有的2D教学图即可。
3. **先跑DR验证全流程** — 不要一次生成5台设备的20张图，DR一台验证通了再铺开。
4. **平面图prompt的共用尾巴必须带上**：白色背景，1:1比例，高质量3D产品渲染，适合教学展示，非写实风格，材质清晰统一。
5. **第2~4步的参考图必须用AI出的正面图**，不能用原始照片——这样才能保证4个角度风格统一。
6. **超声/CT的参考图来源于网络搜索**，实际内容未经人工审核，生成前看一眼确认是正经设备图。
